API Documentation

Hello, hello you beautiful aspiring genius web developer. You'll find on this page the OFFICIAL API Documentation for the IDP Project. To put it simply, every methods you need to acquire not only the best grade evaaar, but also fame and the respect of your peers

State of the API

Here is indicated what is implemented in the API at that point in time and what is soon to be implemented

Implemented

  • Event : Get an all event with stuffs and attendees linked, create, delete, update

  • User : Create, get one by ID, get the events linked to a user

  • Attendee : create, delete, update

  • Stuffs : create, delete, update

To implement

  • Task : integrate Tasks in a similar way as Stuffs

  • User : Create user on the system ready for authentication

  • Social Authentication with Facebook and Google

  • Implement restriction as who can do what (only admin can modify, etc..)

To start

The entry point of the application is : http://clementhamon.com/IDP/public

This might change in the future

If you want to test the request without coding them I'll recommend using something like Postman for Chrome

FYI

If you update an int field to NULL it won't set it to NULL but to 0

If you update a row with the exact same set of values already in it'll likely throw an error

Get event by link

Get the event by is the hash used as a linkId, return the event infos as well as the stuffs, tasks and attendees linked to it

Similar to eventById but this method will be the one with the shared links

This way we prevent people to guess the link to party they are not supposed to join

URL event/link/:eventLink

eventLink : linkId of the event
Method GET
URL Params Required: eventId : string

example: eventLink=44b74bd03e59045dde0afaf60e3a04ff
Success response

Similar to get event by id

Error response

Similar to get event by id

Create event

Create an event and insert it into the database

URL event/new/
Method POST
URL Params Required: name : string

example: eventId="Wild camping trip"

Optionnal:
date : date

example: date=2016-04-01 in yyyy-mm-dd

time : time

example: time=15:00:00 in HH:MM:SS

duration : int

example: duration=120 in minutes

locationName : string

example: duration="Kungshamra 51"

locationLat : decimal

example: locationLat=60.205491

locationLong : decimal

example: locationLong=24.655900

description : string

example: description="Everybody comes, bring your swag, Mohit will teach us the sun dance at sunset"

coverPicture : string

example: description="http://wallpaperus.org/wallpapers/06/211/otters-underwater-2623x1780-wallpaper-946449.jpg"

spotifyPlaylist : string

example: spotifyPlaylist="https://play.spotify.com/user/patpatou/playlist/7fRekMsx0g0T3hm6XUpZwT"
Success response

Code : 200

Content : 
{
  "id": 5,
  "msg": "Event created"
}

id is the id in the database of the newly created event

Error response

Code : 400

Content : 
{
  "error": "Error, new event couldn't be created : you need to specify a name"
}

Code : 400

{
  "error": "Error, new event couldn't be created"
}

Create stuff

Create and insert a new stuff, an event is required but owner is optional

URL stuff/new
Method POST
URL Params Required: name : string

example: eventId="Hookers and chicken tikka"

event : int

example: event=1

The event to which the stuff will be linked

Optionnal: owner : int

example: owner="2039"

ID of the user defined as owner of the stuff
Success response

Code : 200

Content : 
{
  "id": 1,
  "name": "Martini",
  "event": 1,
  "owner": 2039
}

Error response

Code : 404

Content : 
{
  "error": "No Stuff correspond to this id in the database"
}

User

Create user

Create and insert a user into the database, this method is likely to change once the Authentication will be implemented

URL user/new
Method POST
URL Params Required:
firstName : string

example: firstName="Charlie"

lastName : string

example: lastName="Brown"

email : string

example: email="xXx--naruto89killer--xXx@hotmail.fr"

This as to be unique in the database

Optional:
password : string

example: password="hcidlove123"

For now it is store has it is in the database, I'll encode it later on and it will move to required at the same time

avatar : string

example: avatar="https://i.vimeocdn.com/portrait/8748514_300x300.jpg"

If no avatar is specified it's set to this one : http://clementhamon.com/host/avatar-square.jpg
Success response

Code : 200

Content : 
{
  "id": 2043,
  "msg": "User successfully created"
}

Error response

Code : 404

Content : 
{
  "error": "Error, new user couldn't be created : you need to specify a last name, first name and email"
}

Or

{
  "error": "Error, new event couldn't be created"
}
